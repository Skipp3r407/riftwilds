# Rift Storm Anti-Abuse

## Controls

- Server-side eligibility + claim keys (no client-trusted rewards)
- Anti-AFK / login-alone denial
- Diminishing returns on spam actions
- Fraud risk score soft-caps selection weight
- Duplicate wave / SOL / community claim keys
- Regional gate when storm is not global
- Seed commit/reveal for post-event audit
- Emergency cancellation

## Optional SOL

- Flag `RIFT_STORM_SOL_ENABLED` default **false**
- Verify pool balance, per-grant / per-storm / per-user-day caps
- Require wallet; fail → non-SOL Credits substitute
- Record stub signature; real chain send stays off until treasury ops arm it
- Never promise financial return; Community Reward Treasury / promo pool framing only

## Privacy

World banners never force recipient reveal. Opt-in highlights only.
